                              │                     │
                              │   Azure Services    │

                              │                     │
                              └─────────────────────┘

Data Flows:
━━━━━━━━━━━
• Creator → System: Upload media, login, register, edit/delete
• System → Creator: Media list, AI tags, analytics
• Consumer → System: Browse, search, like, rate, comment
• System → Consumer: Media feed, search results
• System ↔ Azure: Store/retrieve data, AI analysis

## 5. Authentication Flow Diagram (JWT)

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                        JWT AUTHENTICATION FLOW                              │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  ┌──────────┐                      ┌──────────────────┐                     │
│  │  Client  │                      │  Azure Functions │                     │
│  │ (Browser)│                      │  (Auth Endpoint) │                     │
│  └────┬─────┘                      └────────┬─────────┘                     │
│       │                                     │                               │
│       │  1. POST /api/auth/register         │                               │
│       │     { name, email, password, role } │                               │
│       │─────────────────────────────────────▶                               │
│       │                                     │                               │
│       │                                     │  2. Hash password (bcrypt)    │
│       │                                     │     Store in Cosmos DB        │
│       │                                     │     Generate JWT token        │
│       │                                     │                               │
│       │  3. Return { token, user }          │                               │
│       │◀─────────────────────────────────────                               │
│       │                                     │                               │
│       │  4. Store token in localStorage     │                               │
│       │                                     │                               │
│  ═════╪═════════════════════════════════════╪════════════════════════════   │
│       │         SUBSEQUENT REQUESTS         │                               │
│  ═════╪═════════════════════════════════════╪════════════════════════════   │
│       │                                     │                               │
│       │  5. POST /api/photos                │                               │
│       │     Header: Authorization: Bearer <token>                           │
│       │─────────────────────────────────────▶                               │
│       │                                     │                               │
│       │                                     │  6. Verify JWT signature      │
│       │                                     │     Extract user info         │
│       │                                     │     Check role (creator?)     │
│       │                                     │                               │
│       │  7. Return { success, data }        │                               │
│       │◀─────────────────────────────────────                               │
│       │                                     │                               │
└───────┴─────────────────────────────────────┴───────────────────────────────┘

JWT Token Structure:
━━━━━━━━━━━━━━━━━━━━
┌─────────────────────────────────────────────────────────────────────────────┐
│  HEADER           │  PAYLOAD                │  SIGNATURE                   │
│  { alg, typ }     │  { id, email, name,     │  HMACSHA256(                 │
│                   │    role, avatar, exp }  │    header.payload, secret)   │
└─────────────────────────────────────────────────────────────────────────────┘
```
